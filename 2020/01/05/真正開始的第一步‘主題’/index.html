真正開始的第一步‘主題’
2020-01-05 01:08   label hexo

系因為感冒而在11點就躺床上了。

我趁這個自己不會忍不住跑去找她玩的時候趕緊沉澱心情來弄一下Hexo的主題，感覺第一步的主題沒弄好，很多東西也就無法繼續下去。

在官網上逛了逛，原本以為隨意借一個簡潔的主題來修改一下就好，然後瞄到了Makito的Typography，不管是字體、標題、排版還是色彩搭配都很合口味。但是，點進去後並不是Typography的主題，而是更簡潔、更讓人喜愛的Journal.。

像是找到了命中之人的那種感覺，原本打開了幾個後備的主題立刻關掉了，官網的主題頁面也關掉。專注於第一次真正的主題克隆。

克隆主題很簡單，跟著Makito的教程很容易就搞定了。

只是心急如焚的我沒認真留意，換好了主題才發現自己換成了Typography。

嘲笑自己一聲，從Makito的Github主頁找到了Journal.的主題，重新再把簡單的步驟再走一次。


First, cd into your Hexo root directory.

這裡我是從自己的文件夾右鍵直接Git Bash。


or npm i --save hexo-renderer-ejs hexo-generator-archive hexo-generator-category-enhance hexo-generator-feed hexo-generator-index hexo-generator-tag for npm users。

這裡Makito是用標註的方式告訴使用npm方法的用戶們如果不是使用yarn的話應該怎樣操作，很貼心。於是我便打上npm install之後copy/paste上面的代碼。


cd themes

完成上一步後cd到themes的文件夾再繼續。


git clone https://github.com/SumiMakito/hexo-theme-Journal.git journal

開始克隆journal的主題到themes文件夾。


cd journal

完成後cd到journal的文件夾再繼續下一步。


yarn install # or npm i for npm users

完成安裝。


找主題花了10分鐘，期間摸魚跑去看視頻看了一個多小時，然後花了5分鐘克隆主題，再花5分鐘記錄一下。

現在踏入凌晨1點，差不多該是時候休息，然後再想想該如何定制化成為自己的主題。


感謝Makitoさん，身處網絡不好擅自推測性別，幸好日文不太存在這個問題。

只是簡單掃了幾眼Github的頁面和博客就覺得Makitoさん是個很厲害的人，希望什麼都半吊子的自己有天能夠達到這樣的水平。
